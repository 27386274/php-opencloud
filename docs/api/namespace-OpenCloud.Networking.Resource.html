Namespace OpenCloud\Networking\Resource

Classes summary
Network

A network is an isolated virtual layer-2 broadcast domain that is typically reserved for the tenant who created it unless you configure the network to be shared. The network is the main entity in the Networking service. Ports (OpenCloud\Networking\Resource\Port) and subnets (OpenCloud\Networking\Resource\Subnet) are always associated with a network.
Port

A port represents a virtual switch port on a logical network switch, represented by OpenCloud\Networking\Resource\Network. Virtual instances (such as servers created using the OpenCloud\Compute\Service) attach their interfaces into ports. The port also defines the MAC address and the IP address(es) to be assigned to the interfaces plugged into them. When IP addresses are associated to a port, this also implies the port is associated with a OpenCloud\Networking\Resource\Subnet, as the IP address is taken from the allocation pool for a specific subnet.
Subnet

A subnet represents an IP address block that can be used to assign IP addresses to virtual instances (such as servers created using the OpenCloud\Compute\Service. Each subnet must have a CIDR and must be associated with a network.
